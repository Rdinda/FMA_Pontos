# Política de Segurança

## Versões suportadas

Este projeto é um aplicativo Flutter e publica versões via GitHub Releases. Atualizações de segurança são fornecidas apenas para a versão mais recente disponível na série atual.

| Versão | Suporte de segurança |
| ------ | -------------------- |
| 1.0.x  | Sim                  |
| < 1.0  | Não                  |

## Reportando uma vulnerabilidade

Não abra issues públicas para reportar vulnerabilidades.

Para reportar, use o fluxo de reporte privado do GitHub:

1. Acesse a aba **Security** do repositório.
2. Clique em **Report a vulnerability**.
3. Envie os detalhes do problema.

Inclua, se possível:

- Versão do app (ex.: 1.0.16)
- Plataforma e versão do SO (Android/iOS)
- Passo a passo para reproduzir
- Impacto esperado e impacto observado
- Evidências (logs, prints), sem dados sensíveis

Prazos:

- Confirmação de recebimento: até 72 horas
- Atualizações de status: conforme avanço da triagem e correção

Se a vulnerabilidade for aceita, vamos coordenar a correção e a publicação de uma versão com patch. Se for recusada (por exemplo, comportamento esperado ou não reproduzível), explicaremos o motivo.
